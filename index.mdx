## How GX Core works

All GX workflows follow a simple four-step pattern:

<Steps>
  <Step title="Set up your environment">
    Install GX Core and create a **Data Context** — the central object that manages all your GX configuration, metadata, and results.

    ```bash
    pip install great_expectations
    ```

    ```python
    import great_expectations as gx
    context = gx.get_context()
    ```
  </Step>
  <Step title="Connect to your data">
    Add a **Data Source** pointing to your data store, define a **Data Asset** (a table, file, or query result), and create a **Batch Definition** to control how records are retrieved for validation.

    ```python
    data_source = context.data_sources.add_pandas_filesystem(
        name="my_data_source",
        base_directory="./data",
    )
    data_asset = data_source.add_csv_asset(name="yellow_tripdata")
    batch_definition = data_asset.add_batch_definition_whole_dataframe(
        "all_records"
    )
    ```
  </Step>
  <Step title="Define Expectations">
    Create an **Expectation Suite** and add **Expectations** — verifiable assertions about your data — to describe what you expect.

    ```python
    suite = context.suites.add(gx.ExpectationSuite(name="my_suite"))
    suite.add_expectation(
        gx.expectations.ExpectColumnValuesToNotBeNull(column="passenger_count")
    )
    suite.add_expectation(
        gx.expectations.ExpectColumnValuesToBeBetween(
            column="trip_distance", min_value=0, max_value=1000
        )
    )
    ```
  </Step>
  <Step title="Run Validations">
    Create a **Validation Definition** linking your data to your suite, then run it to get a **Validation Result**. Use a **Checkpoint** to automate this in production.

    ```python
    validation_definition = context.validation_definitions.add(
        gx.ValidationDefinition(
            name="my_validation",
            data=batch_definition,
            suite=suite,
        )
    )
    results = validation_definition.run()
    print(results)
    ```
  </Step>
</Steps>
